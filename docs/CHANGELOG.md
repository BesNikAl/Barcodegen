# CHANGELOG

Все изменения проекта фиксируются в этом документе.

---

## v0.3.0

### Добавлено
- Каркас приложения BarcodeGen.
- Генерация DataMatrix ECC200.
- Предпросмотр DataMatrix.
- Масштабирование изображения.
- История с сохранением в history.json.
- Dark Theme.
- InputPanel.
- PreviewWidget.
- HistoryManager.

### Известные проблемы
- Отображение DataMatrix выполняется по двойному клику по строке истории.
- Редактирование данных в таблице не обновляет history.json и предпросмотр.
- Некорректное изменение ширины столбцов таблицы истории.
- Validator ещё не вынесен в отдельный класс.

### Следующий этап
1. Перевести историю на одинарный выбор строки.
2. Добавить HistoryManager.update().
3. Обновление истории после редактирования.
4. Исправить HeaderView.
5. Вынести Validator.

## v0.3.1

- В таблице истории колонка «Дата» заменена на колонку «Комментарий».
- Добавлена поддержка хранения комментариев для записей истории.
- Для совместимости существующее поле даты сохранено во внутреннем формате данных.

## v0.4.1

- Добавлен импорт истории из CSV.
- Поддерживаются CSV с одной колонкой (data) и двумя колонками (data, comment).
- При импорте автоматически пропускаются дубликаты и некорректные записи.
- После импорта история и интерфейс обновляются автоматически.

## v0.4.2

- В интерфейс добавлена кнопка «Импорт CSV».
- Подготовлена инфраструктура для реализации импорта истории из CSV.

## v0.4.3

- Добавлен модуль core/csv_importer.py.
- Реализовано чтение CSV с автоматическим определением разделителя.
- Поддерживаются форматы с одной и двумя колонками.
- Добавлена поддержка UTF-8 и UTF-8 BOM.

## v0.4.4

- Реализован импорт записей из CSV через диалог выбора файла.
- Импортированные записи автоматически сохраняются в историю.
- После завершения импорта таблица истории обновляется, а в строке состояния отображается статистика импортированных и пропущенных записей.

## v0.4.5

- Сборка проекта переведена на использование BarcodeGen.spec.
- build.bat упрощён и теперь использует единую конфигурацию PyInstaller.
- Исключено дублирование параметров сборки между build.bat и spec.
- Определены правила использования build.bat.
- Зафиксирована структура итоговой папки приложения для распространения.
